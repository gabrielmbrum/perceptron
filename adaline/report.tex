\section{Introduction}
The ADALINE (Adaptive Linear Element) is a single-layer neural network that uses the Delta Rule for weight updates. Unlike the Perceptron, the error is calculated based on the continuous activation potential ($u$), allowing for the minimization of the Mean Squared Error (MSE) through gradient descent. This report analyzes the behavior of ADALINE using two optimization approaches: Stochastic Gradient Descent (SGD) and Batch Gradient Descent. Crucially, this analysis investigates a specific phenomenon observed during testing: how defining the stopping criterion by the absolute difference between the current and previous error ($|E_t - E_{t-1}| \le \epsilon$) can trigger premature early stopping, preventing the model from achieving optimal test results.

\subsection{Discussion of Results}
Despite the data being linearly separable, the stopping criterion severely truncated the learning process. 

\textbf{Stochastic Gradient:} Training stopped prematurely at epoch 6 due to the precision criterion. While the model achieved a high accuracy of 95.00\% on the test set, it still recorded 6 misclassifications in the training set at the final epoch. This indicates that stopping based on the immediate error difference prevented the algorithm from finding the perfect linear separation.

\textbf{Batch Gradient:} Exhibited an even more drastic early stop at epoch 2. It achieved the same 95.00\% test accuracy but with 7 training misclassifications. Because the Batch method averages the gradients, the error descent is smoother; thus, the difference between epoch 1 and 2 was small enough to trigger the precision threshold almost immediately, halting convergence.

\subsection{Discussion of Results}
In this dataset, the presence of noise creates overlapping regions between classes.

\textbf{Stochastic Gradient:} The noisy nature of this dataset caused the SGD to trigger the early stopping condition at just epoch 2. This yielded an accuracy of 61.33\% on the test set with 83 misclassifications in training. The sample-by-sample update combined with noise caused immediate error fluctuation that mistakenly signaled convergence to the stopping function.

\textbf{Batch Gradient:} The batch approach accumulated the gradients, making it slightly more robust to local noise fluctuations and bypassing the premature stop of the initial epochs. It managed to train until epoch 40 before the precision criterion triggered. However, the final test accuracy was slightly lower (60.00\%) with 87 misclassifications.

\subsection{Discussion of Results}
In a high-dimensional scenario, the choice of hyperparameters clearly exposed the flaws of the relative error stopping criterion.

\textbf{Impact of Precision ($\epsilon$):} When using a looser precision ($\epsilon = 0.1$), the model stopped too early across both methods. For instance, with $\eta = 0.001$, SGD stopped at epoch 9 yielding only 77.78\% accuracy. By tightening the precision to $\epsilon = 0.0001$, the model was forced to train until epoch 35, which significantly improved the test accuracy to 87.30\%. 

\textbf{Method Comparison:} Overall, SGD achieved the highest peak accuracy (87.30\%) compared to Batch (85.71\%). Batch training struggled particularly with larger learning rates ($\eta = 0.01$), stalling at 66.67\% accuracy. This reinforces that while Batch is more stable, SGD's ability to navigate complex error surfaces allows it to find better minima in high-dimensional spaces, provided the stopping criterion does not halt it prematurely.

\section{Conclusion}
This activity demonstrated the behavior of the ADALINE network under different optimization techniques. The most critical finding of this report is that utilizing the absolute difference between consecutive epoch errors ($|E_t - E_{t-1}|$) as a stopping criterion is a flawed approach. As observed across all three datasets, the learning curve can flatten temporarily or descend slowly, triggering a stop long before the global minimum is reached. 

While Stochastic Gradient Descent generally achieved better peak accuracies than Batch Gradient Descent in high-dimensional spaces, both methods were heavily bottlenecked by premature stopping. Future implementations should consider alternative stopping criteria, such as evaluating the error on a separate validation set or using a "patience" threshold that waits for multiple epochs of non-improvement before halting the training.
